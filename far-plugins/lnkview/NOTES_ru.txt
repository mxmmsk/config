Версия 3.4.0
------------

1. Реализован просмотр элементов LNK-файла в HEX-формате при нажатии клавиши F3 на виртуальном
   файле или директории в панели плагина (за основу HEX-viewer'а взят движок из плагина Hexitor).
   В окне HEX-viewer'а клавиша F6 влючает/отключает показ форматированных данных под курсором.
   При смене кодовых страниц (по клавише F8) кодовая страница 1200 (unicode) показывается дважды -
   один раз с четным смещением начала символов, другой - с нечетным.

2. Расширена внутренняя база данных по GUID-ам за счет включения информации из PortableDevice.h
   Соответственно, при просмотре ItemIDList-ов, относящихся к WPD (Windows Portable Device), для
   абсолютного большинства Property Keys теперь показываются их мнемонические имена. 



Версия 3.3.0
------------

1. Команда lnk:create, введенная без указания параметра <folder name> теперь пытается 
   создать LNK-файл в АКТИВНОЙ панели, если ПАССИВНАЯ панель закрыта или не является обычной файловой панелью.

2. Команда lnk:create теперь отписывает в создаваемый LNK-файл параметры командной строки 
   для запуска целевого файла (см. хелп плагина).

3. Теперь команда lnk:follow отображает целевой файл на АКТИВНОЙ панели, если ПАССИВНАЯ панель закрыта.

4. Путь по умолчанию к директории "Избранных ссылок" в диалоге конфигурации изменен на %FARLOCALPROFILE%\..\FavLinks.

5. В диалог конфигурации добавлена опция "Включать путь в имя создаваемого LNK-файла".
   Если опция отключена, команда lnk:create при создании LNK-файлов не включает в имя создаваемого файла
   полный путь до таргет-объекта.



Версия 3.2.0
------------

1. В команде lnk:cat теперь можно не задавать имя выходного файла.
   В этом случае команда экспортирует данные во временный файл и открывает его во внутреннем Viewer-е FAR'а.
   Если не задан и первый параметр (имя LNK-файла), команда  экспортирует данные из текущего 
   элемента АКТИВНОЙ панели во временный файл и открывает его во внутреннем Viewer-е FAR'а.

2. Все поля, содержащие время, теперь показываются с максимальной точностью (до 100 наносекунд).

3. В меню плагинов (по кнопке F11) теперь показывается команда плагина, которая будет выполнена 
   при вызове плагина из меню плагинов.

Версия 3.1.0
------------

1. Команда lnk:cat теперь может принимать "*" в качестве первого параметра.
   Это позволяет экспортировать данные сразу из нескольких LNK-файлов, выбранных на АКТИВНОЙ панели,
   в файл csv-формата, пригодный для загрузки в MS Excel или любой иной SpreadSheet для последующей обработки.

   Darwin Descriptors в соответствующей секции XtraData, а также в PIDL extension 0xbeef000b.

   HKCU\SoftWare\Microsoft\Windows\CurrentVersion\Explorer\StartPage\ProgramsCache
   (см. хелп плагина)


 
Версия 3.0.0
------------

1. Появилась команда lnk:follow, которая позволяет найти и отобразить на 
   ПАССИВНОЙ панели целевой объект заданного LNK-файла.
   Если эту команду назначить на клавишу, скажем, <Ctrl-PgDn> (в диалоге конфигурации), 
   то при нажатии этой клавиши на LNK-файле можно "переходить по ссылке" в один клик. 

2. Появилась команда lnk:create, которая позволяет создавать LNK-файлы.
   В том числе, позволяет создавать ссылки на выбранные файлы активной панели.
   По умолчанию LNK-файлы создаются в текущей директории пассивной панели.
   Если назначить эту команду на вызов из меню плагинов, можно "оптом" создавать 
   ссылки на выбранные файлы за пару кликов.

   ВНИМАНИЕ:
   Данная команда позволяет создавать LNK-файлы, ссылающиеся на другие LNK-файлы 
   (что невозможно сделать обычными средствами через интерфейс MS Shell).
   Когда такая цепочка косвенных ссылок обрабатывается Shell-ом, открывается 
   целевой объект конечного LNK-файла.
   Будьте осторожны - НЕ СОЗДАВАЙТЕ ЦИКЛИЧЕСКИХ ЦЕПОЧЕК LNK-ФАЙЛОВ.

3. Появилась возможность вызова плагина из меню дисков.
   При этом открывается директория "любимых ссылок" (имя которой, разумеется,
   задается в диалоге конфигурации плагина).
   Эту же директорию можно открыть командой lnk:fav.
   Наличие такой директории в сочетании с lnk:follow, назначенной на <Ctrl-PgDn>,
   позволяет быстро перескакивать на нужные места в файловой системе.

   Создавать "любимые ссылки" можно с помощью lnk:create, когда директория 
   "любимых ссылок" открыта на ПАССИВНОЙ панели, но удобнее это делать без 
   переключения панелей командой lnk:addfav, которая, как и lnk:create,
   может быть назначена на вызов из меню плагинов.

4. Команда lnk:bagmru теперь может экспортировать данные из отдельного hive-файла реестра.
   При этом файл реестра открывается "только на чтение" и обрабатывается вручную.
   Административных привиллегий для такой операции не требуется.

5. Для поддержки новых возможностей переработан диалог конфигурации плагина.    

   Подробное оптсание всех перечисленных команд см. в хелпе плагина
